# 设计脑暴（必须项）

时间：2026-02-20

## 方案A：iOS Utility Minimalism（概率0.42）
- **设计流派**：iOS 风格的工具型极简主义
- **核心原则**：
  1) 仪表盘式信息优先级（数字/进度条）
  2) 大留白 + 轻描边卡片
  3) 单一强 CTA（蓝色主按钮）
  4) 动效克制但明确（按钮按压、卡片选中）
- **色彩哲学**：冷静的浅灰白背景 + 高对比蓝色作为“行动色”，传达“效率、专注、可控”。
- **布局范式**：非居中落地页；采用“移动端竖屏容器”+ 桌面端居中展示手机框的方式模拟 App。
- **标志元素**：
  - 顶部搜索条
  - 大数字统计卡
  - 蓝色圆角主按钮
- **交互与动效**：
  - 进入背词页：卡片从下向上弹出（轻微）
  - 选项卡：选中有描边/阴影变化
  - 答题反馈：正确绿色、错误红色并抖动
- **字体体系**：
  - Display：Ma Shan Zheng（标题/数字强调）
  - Body：Noto Sans SC（正文）

## 方案B：Paper Study Desk（概率0.24）
- **设计流派**：文具/纸张拟物的学习桌氛围
- **核心原则**：
  1) 纸张纹理背景
  2) 贴纸式选项
  3) 手写感标题
- **色彩哲学**：米白 + 铅笔灰 + 书签红
- **布局范式**：不规则错位卡片、像贴在桌面上的便签
- **风险**：拟物过强可能偏“装饰”，不够像百词斩

## 方案C：Neo-Brutal Exam Mode（概率0.18）
- **设计流派**：新粗野主义（强对比、硬边）
- **核心原则**：
  1) 强对比色块
  2) 大标题 + 粗线框
  3) 强烈反馈
- **色彩哲学**：黑白底 + 高饱和蓝/黄
- **风险**：与百词斩既有“清爽”气质不一致

## 选择与承诺

选择：**方案A：iOS Utility Minimalism**。

原因：用户明确希望“尽量复刻百词斩的界面样式”，方案A最贴近公开截图中的信息结构与视觉语言，同时能在 Web 端做出“像 App”的体验。
